Step 1 — Hint Detection

wantsBuilderTools(message) scan หาคำสำคัญใน message
ถ้าเจอ → includeBuilder = true → ส่ง builder tools ไปกับ request
⚠ เฉพาะ action verbs — nouns ("dashboard","chart","กราฟ") ถูกเอาออกแล้ว เพราะทำให้เกิด false positive เช่น "อธิบายกราฟ" ไม่ควรเป็น builder intent

🇬🇧 English — action verbs only
createaddremove deletebuildmake set upsetupchange editrenameupdate ackacknowledgeresolve
🇹🇭 ภาษาไทย — action verbs only
สร้างเพิ่ม ลบแก้ไข เปลี่ยนปรับ ตั้งค่าอัปเดต อัพเดทยืนยัน ลบออก
Force true เสมอ ถ้า:
✓ context มีค่า (มี dashboard อยู่บนหน้าจอ)
⚠ Fallback: hint miss แต่ model เรียก builder tool → Groq ตอบ error "not in request.tools" → code retry อัตโนมัติด้วย full tools

หลัง iteration แรก callTools = nil → iteration ถัดไปไม่ส่ง tools ซ้ำ

RBAC — Role-Based Access

enforce 2 ชั้น: (1) ไม่ส่ง tool schema ไปให้ model และ (2) dispatch() check role อีกครั้งก่อน execute จริง

viewer
✅ อ่าน telemetry, alerts, dashboards
❌ ไม่สามารถสร้าง / แก้ไข / ลบ ใดๆ
editor
✅ ทุกอย่างของ viewer
✅ add/remove widget, create alert
✅ ack / resolve alert event
admin
✅ ทุกอย่างของ editor
✅ สร้าง dashboard ใหม่ (via confirm button)
✅ manage ทุก alert

Machine Name Resolution

ค้นหายังไง?
ใช้ SQL LOWER(name) LIKE '%' || LOWER($input) || '%'
→ partial match ได้ เช่น "cw01" เจอ "CW-01"
→ exact match ได้ priority สูงกว่า partial match
→ org-scoped เสมอ (ผ่าน factory → production_line → machine)

Step 3 — Agentic Loop (max 5 iterations)

1
Build messages array:
system_prompt + history (8 messages ล่าสุด จาก DB) + context (dashboard บนหน้าจอ ถ้ามี)
⚠ tool messages (role="tool") ไม่ถูกใส่ใน history — มีแค่ user + assistant เท่านั้น
2
Call Groq API พร้อม tools (iteration แรก) หรือไม่มี tools (iteration ถัดไป)
→ รอ response จาก qwen3-32b
3
ถ้า finish_reason = "tool_calls" → execute tool ผ่าน dispatch()
→ บันทึก tool message ลง DB (role="tool")
→ append tool result กลับเข้า messages array
→ loop ต่อ (iteration ถัดไปไม่ส่ง tools)
4
ถ้า finish_reason ≠ "tool_calls" → บันทึก assistant message ลง DB
→ return response กลับ frontend
✗
ถ้าครบ 5 iterations แล้วยังไม่จบ → exit loop โดยไม่มี response text

Retry Logic (3 paths)

PATH 1
error: "not in request.tools"
hint scan miss แต่ model เรียก builder tool → Groq reject
→ set includeBuilder = true → retry พร้อม full tools (1 ครั้ง)
PATH 2
error: "Tool choice is none"
qwen3 reasoning model พยายาม chain tools ใน summary call (ไม่มี tools ใน request)
→ retry พร้อม original tools array (qwen3-specific bug)
PATH 3
error: "Failed to call a function"
model generate malformed tool call → Groq parse ล้มเหลว
→ retry โดยไม่ส่ง tools เลย (plain text fallback)
RATE LIMIT
HTTP 429 Too Many Requests
อ่าน Retry-After header (max 8s) → sleep → retry สูงสุด 3 ครั้ง

Token Cost Optimizations

🔍 Hint-based tool filter
ส่งแค่ tools ที่จำเป็น — ไม่ส่ง builder tools สำหรับ Q&A ธรรมดา (~8 tools แทน 14)
🗂 History cap 8 messages
ตัด history เก่า เก็บแค่ 8 messages ล่าสุด (user + assistant เท่านั้น ไม่รวม tool messages)
🚫 Drop tools after iter 1
callTools = nil หลัง iteration แรก — iteration ถัดไปไม่ส่ง tool array ซ้ำ (ประหยัด ~14k tokens)
🙈 reasoning_format: hidden
ซ่อน thinking tokens ไม่ให้อยู่ใน output — ลด output tokens จาก qwen3 reasoning
📍 Context injection (conditional)
ส่ง dashboard context เฉพาะเมื่อมี (on-screen state) inject หลัง history เสมอ (recency wins)
⚡ /no_think flag
ต่อท้าย system prompt → บอก qwen3 ไม่ต้อง think ยาวสำหรับ Q&A ธรรมดา
