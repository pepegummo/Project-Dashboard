Factory Knowledge Graph — injected as system message before every turn

What gets injected (example org):

What the AI resolves from graph (example query):

User says
"how many pieces in range 100–200 today?"
pieces → CW-01.output (pcs/hr)
range 100–200 → highlight_min:100, highlight_max:200
today → time_range:"24h"
widget type → line-chart (trend over time)
Without graph: AI would ask "which machine?", "which field?", "what unit is pieces?"
Tool Registry

All tools — Swagger-style. SelectTools(graph, role) picks the right subset.

get_telemetry_trend
existingviewer
Get avg/min/max of one metric over a time window. Primary tool for range queries and trend visualisation.
Parameters
machine_idstringrequired
metricstringrequired
time_rangeenum5m 15m 30m 1h 6h 24h 7d 15d 30d
gate: always → line-chart
get_latest_telemetry
existingviewer
Get current live readings for one machine. Used when user asks "what is X right now?"
Parameters
machine_idstringrequired
gate: always → gauge / kpi-card
get_daily_count
existingviewer
Per-day production count for one machine. Used for "how many pieces this week / today?"
Parameters
machine_idstringrequired
daysintegerdefault 7
gate: always → daily-count
get_active_alerts
existingviewer
List all open alert events. Used for "any alarms?", "what's wrong?", "is X normal?"
Parameters
(none)—org-scoped by JWT
gate: always → alarm-panel
get_machines
existingviewer
List all machines with field keys. Rarely needed — graph context already contains this. AI calls only when question is ambiguous across machines.
Parameters
(none)—org-scoped by JWT
gate: always → text only
answer_with_widget
★ NEWviewer
Package a text answer with a widget spec. AI calls this LAST after fetching data — it tells the frontend what to render alongside the text bubble.
Parameters
textstringrequiredAnswer to show in chat bubble
widget.typeenumrequiredline-chart · gauge · kpi-card · status-card · daily-count
widget.machinestringrequiredMachine name (resolved to UUID in dispatch)
widget.metricstringrequiredField key e.g. "output"
widget.time_rangeenum5m 15m 30m 1h 6h 24h 7d
widget.highlight_minnumberLower bound of range to shade on chart
widget.highlight_maxnumberUpper bound of range to shade on chart
widget.titlestringWidget header label
widget.unitstringOverride display unit
gate: always → any widget type (inline in chat)
