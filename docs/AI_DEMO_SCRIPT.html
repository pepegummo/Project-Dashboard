🎬 Scenarios

1 Greeting — language & personality ~15 sec no tool
🗣 Say & type

Type exactly:

สวัสดีครับ ช่วยอะไรได้บ้าง?

("Hello, what can you help me with?")

🖱 Actions
  • Press Enter
  • Wait ~1 second for response
👁 Expected result
A friendly Thai-language greeting appears as a chat bubble. No widget card — just text. Response is < 1 second.
💡 Talking point: Thai works out of the box. No language setting, no prefix. The AI replies in whatever language you write in. Greetings cost almost nothing — only ~273 prompt tokens, no tools sent to Groq.
2 Live metric — instant widget card ~30 sec show_metric
🗣 Say & type

Type:

แสดง speed ของ CW-01 ให้หน่อย

("Show me the speed of CW-01")

Then follow up:

เป็น gauge ได้ไหม

("Can it be a gauge?")

🖱 Actions
  • Send first message
  • Point to the live KPI card that appears
  • Note the value is updating in real time
  • Send the follow-up
👁 Expected result
A live KPI card renders in the canvas with the current speed value. After follow-up: a gauge dial card replaces it, needle pointing at the live reading.
💡 Talking point: The AI doesn't just answer — it renders a live widget. The value is fetched fresh from the database, not from the chat history. The follow-up shows multi-turn context: the AI knows you mean the same machine and metric.
3 Read chart data — what is the line chart showing? ~45 sec get_telemetry_series
🗣 Say & type

Type:

temperature ของ CW-01 ใน 6 ชั่วโมงที่ผ่านมาเป็นยังไงบ้าง มีจุดไหนผิดปกติไหม

("What does CW-01 temperature look like in the last 6 hours? Any anomalies?")

🖱 Actions
  • Send message
  • Wait for response (~2 sec)
  • Point to the time-bucketed values in the reply
👁 Expected result
AI replies in Thai with the trend — actual avg/min/max per 5-min bucket for the last 6h. Calls out any spike or dip above/below normal range. Plain text, no widget card.
💡 Talking point: The AI reads the same data the line chart plots — not just a summary average. It gets every bucket so it can reason about when exactly a spike happened. This is the get_telemetry_series tool added in the latest session.
4 Production count — bucket-level output ~30 sec get_production_count
🗣 Say & type

Type:

CW-01 ผลิตชิ้นดีได้เท่าไหร่ใน 24 ชั่วโมงที่ผ่านมา แบ่งเป็น bucket 1 ชั่วโมง

("How many good pieces did CW-01 produce in the last 24h, broken into 1-hour buckets?")

🖱 Actions
  • Send message
  • Point to the 24 rows of hourly counts
  • Note the status filter ("good" only)
👁 Expected result
AI replies with 24 hourly rows of good-piece counts — same shape as the DailyCount widget. May summarize the best and worst hour.
💡 Talking point: Flexible bucketing with status filter. You can ask for any bucket size (1m, 30m, 1h, 1d) and filter by good / reject / all. The AI understands "ชิ้นดี" (good pieces) and passes status:"good" to the query.
5 Active alerts — what's firing right now ~20 sec get_active_alerts
🗣 Say & type

Type:

ตอนนี้มี alert อะไรบ้างที่ยังเปิดอยู่

("What alerts are currently open?")

🖱 Actions
  • Send message
  • Point to severity colors in the reply
👁 Expected result
Plain-text list of open alert events with machine name, metric, value, severity, and trigger time. If none: "ไม่มี alert ที่เปิดอยู่ตอนนี้".
💡 Talking point: Read-only for alerts by design. The AI can list alerts but cannot acknowledge or resolve them — that stays in the Alerts page. This is a deliberate safety boundary, not a limitation.
6 Create a dashboard — preview → edit → confirm ~2 min preview_dashboard · preview_add_widget · preview_update_widget · create_custom_dashboard
🗣 Say & type

Step 1

สร้าง production dashboard สำหรับ CW-01 ให้หน่อย

("Create a production dashboard for CW-01")

Step 2 (after preview appears)

เพิ่ม gauge สำหรับ weight เข้าไปด้วย

("Add a weight gauge too")

Step 3 (after preview updates)

Click Confirm button on the canvas card.

🖱 Actions
  • Send Step 1 — wait for canvas preview to appear
  • Point to each widget in the preview
  • Send Step 2 — watch preview update
  • Point to the new weight gauge in preview
  • Click Confirm
  • Click the link in the success card to open the live dashboard
👁 Expected result
Step 1: Canvas shows a preview card with 4–5 production widgets (count, speed, KPI cards). Confirm + Discard buttons visible.
Step 2: Preview updates — weight gauge added. AI confirms in Thai.
Step 3: Success card with a link. Dashboard exists in the Dashboards list with live widgets.
💡 Talking point: Preview-then-confirm flow — the AI cannot accidentally create. The database write only happens when the user clicks Confirm. The LLM never touches the DB directly. Step 2 shows in-preview editing across multiple turns with no extra navigation.
7 Edit existing dashboard — add widget by name ~45 sec add_widget_to_dashboard
🗣 Say & type

Type:

เพิ่ม line chart สำหรับ temperature ของ CW-01 เข้าไปใน CW-01 Overview

("Add a temperature line chart for CW-01 to the CW-01 Overview dashboard")

🖱 Actions
  • Send message
  • Click the dashboard link in the success card
  • Point to the new widget at the bottom of the grid
👁 Expected result
Success card appears with a link to "CW-01 Overview". Opening the dashboard shows a new line-chart widget for temperature added to the grid.
💡 Talking point: No preview needed for existing dashboards — add_widget_to_dashboard writes directly. The dashboard name is matched by substring so "CW-01 Overview" finds "Machine Overview — CW-01". Editors and admins only; viewers see a permission error.
