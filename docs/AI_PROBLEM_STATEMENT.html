🏭 The Situation

Context
A factory floor runs dozens of machines. Every machine streams live telemetry — speed, temperature, weight, pressure, piece counts. All of it lands in IotVision dashboards. Operators, supervisors, and engineers need fast answers from that data — every shift, every day.
The gap
Getting an answer still requires knowing where to look, navigating to the right dashboard, and reading the right widget. For non-technical staff on the floor, that's a barrier. For engineers with 20 things open, it's friction.

😤 Pain Points

🗺️
Navigation takes time
To check one metric, a user must navigate to the right dashboard, find the right widget, and wait for it to load — even if the question is simple.
ต้องไปหาเองทุกครั้ง ไม่รู้ว่า dashboard ไหน
🔧
Dashboard creation requires expertise
Building a new dashboard means using the drag-and-drop editor, knowing widget types, and configuring each one manually — a task only engineers attempt.
อยากดู overview ของเครื่องใหม่ แต่ไม่รู้จะสร้างยังไง
🌐
Language barrier
The interface is in English. Most operators on the factory floor think and communicate in Thai. Technical terms like "telemetry" and "aggregate" create friction.
ไม่เข้าใจ field name เป็นภาษาอังกฤษทั้งหมด
📊
Cross-metric questions have no single answer
"Was anything abnormal on CW-01 last hour?" requires checking multiple widgets across potentially multiple dashboards — there is no single place to ask.
ต้องไล่ดูทีละ widget ไม่มีที่ถามรวม
🧤
Hands-free environment
Factory floor workers often wear gloves, operate machinery, or stand at a distance from a monitor. Complex UI interactions are impractical mid-shift.
ใส่ถุงมืออยู่ กด widget เยอะไม่ได้

💡 The Solution

What we built
A conversational AI assistant built directly into IotVision. Ask any question about your factory in plain language — get a live widget, a trend summary, or a full dashboard — without touching the configuration UI.
Works in Thai and English. Enforces the same role-based access as the rest of the platform. Backed by Groq (openai/gpt-oss-20b) — fast enough for a factory floor, accurate enough for production data. Every answer is fetched live from the database — the AI never guesses a sensor reading.
