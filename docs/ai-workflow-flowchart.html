Routing Graph Breakdown

  • 1. Token Scan Backend matches names against machines and terminology synonyms (Thai and English) in metric_synonyms.
  • 2. Entity Connection PostgreSQL query joins the selected entity categories to find active tool connections (e.g. `machine` and `metric` trigger read/write telemetry tools).
  • 3. Token/Cost Savings Unrelated widgets, dashboard templates, and alert tools are omitted from the Swagger-like dynamic schema list, optimizing prompt size.

Execution Lifecycle

  • 1. Read Stage The AI queries the database range via get_telemetry_trend, converting the relative term "Pieces" to the database field key inspected.
  • 2. Write Stage (Local Preview) The AI adds a KPI widget to the dashboard context, setting the absolute date ranges so the widget displays the query timeframe.
  • 3. Client-Side Update The frontend local state mounts the preview widget, executing telemetry queries dynamically for the visual card.
