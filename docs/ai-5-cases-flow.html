Routing Graph Query

Since no machines, alerts, or widgets are specified, PostgreSQL connection edges restrict toolsets to general read tools, minimizing payload sizes.

LLM Decision

The model adheres to Rule 1 (plain text answers for greetings) and directly formats a friendly conversation response without triggering functions.

Case 2: Dashboard Preview & Greeting

Final Canvas State

Case 3: Dynamic Preview Customization

Query 1: "create dashboard about cw-01" → Query 2: "change metric from speed to vibration" Preview Context: Present (Turn 2) Entities: ['dashboard', 'widget', 'metric']
sequenceDiagram autonumber actor User as "Factory Operator (User)" participant FE as "Frontend Canvas" participant BE as "Go Backend Server" participant DB as "PostgreSQL DB" participant Groq as "Groq AI Engine" note over User, Groq: Turn 1: Preview dashboard shown (contains widget "Speed Gauge" using metric "speed") User->>BE: "change metric from speed to vibration" BE->>DB: Scan terms: Matches 'vibration' (key: 'vibration') DB-->>BE: Entity categories detected: ['metric', 'widget', 'dashboard'] BE->>Groq: Call Groq with tool schemas & active widgets in context note over Groq: Groq matches widget "Speed Gauge" on screen.
Calls preview_update_widget. Groq->>BE: Tool Call: preview_update_widget(widget_title:"Speed Gauge", metric:"vibration") BE-->>BE: Validates vibration exists on CW-01 BE-->>Groq: Return: { updated: true, changes: { metric: "vibration" } } Groq-->>BE: "I have updated the widget to show vibration." BE-->>FE: Return Message + toolResult changes FE->>FE: Object.assign() applies new metric to local preview

Dynamic Target Mapping

The AI compares the requested edit (vibration) against the active widgets in the context payload, automatically discovering that "Speed Gauge" needs the update.

Zero Database Writes

All updates take place in local Vue state variables (Preview Draft) until the operator explicitly clicks "Confirm" to save.

sequenceDiagram autonumber actor User as "Factory Operator (User)" participant BE as "Go Backend Server" participant Groq as "Groq AI Engine" User->>BE: "change metric from speed to vibration" note over BE: No active preview card on screen (context is empty) BE->>Groq: Call Groq with full builder tools (matched via entities) note over Groq: Groq sees request to change a widget.
But no dashboard context exists to modify.
Cannot target any widget. Groq-->>BE: Returns: "Which widget or dashboard are you looking to customize? Please specify the dashboard name or CW-01 to start a preview plan." BE-->>User: Ask for clarification

Context Check

Without on-screen widgets passed in the context payload, the model is unable to resolve widget titles, preventing accidental tool executions on unselected dashboards.

Fallback Behavior

The AI gracefully asks the operator to name the dashboard or begin a new template session to create context.

Case 5: Explain On-Screen Widget Data

Query: "explain data in widget Trend now" Dashboard Context: Active widgets on screen Entities: ['widget', 'machine', 'metric']
sequenceDiagram autonumber actor User as "Factory Operator (User)" participant BE as "Go Backend Server" participant DB as "PostgreSQL DB" participant Groq as "Groq AI Engine" User->>BE: "explain data in widget Trend now" note over BE: Context contains widget "Trend" (Machine: CW-01, Metric: speed) BE->>DB: Query tools connected to ['machine', 'metric'] DB-->>BE: Returns ['get_latest_telemetry', 'get_telemetry_trend'] BE->>Groq: Call Groq with telemetry tools + context note over Groq: Groq resolves "Trend" = CW-01 speed.
Calls get_latest_telemetry. Groq->>BE: Tool Call: get_latest_telemetry(machine_id: "CW-01") BE->>DB: SELECT data->>'speed' FROM telemetry_raw WHERE machine_id=CW-01 LIMIT 1 DB-->>BE: Returns: 1200 BE-->>Groq: Return: { values: { speed: 1200 } } Groq-->>BE: Returns: "Currently, CW-01's speed is averaging 1200 rpm, which matches normal line throughput." BE-->>User: Display plain text explanation

Semantic Association

The AI resolves the informal name "Trend" to the database metric identifier speed and machine identifier CW-01 using the active canvas widget definitions.

Telemetry Retrieval

The AI calls telemetry read APIs dynamically, presenting raw database records as a natural language summary to the operator.
