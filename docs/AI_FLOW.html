🔀 Sequence Diagram

sequenceDiagram
    actor User
    participant B as Browser
    participant API as Backend API
    participant Groq as Groq LLM
    participant DB as TimescaleDB

    User->>B: Types message (Thai / English)
    Note over B: Attach context: live widget values · dashboard state

    B->>API: POST /api/ai/chat

    API->>DB: Save user message · load last 8 messages
    DB-->>API: Conversation history

    API->>Groq: System prompt + history + tools (filtered by role)

    loop Agentic loop — up to 5 rounds
        Groq-->>API: tool_call (e.g. show_metric)
        Note over API: Role check — viewer blocked from write tools
        API->>DB: Query telemetry / machines / dashboards / alerts
        DB-->>API: Result data
        API->>Groq: Tool result appended to thread
        Note over Groq: Round 2+: tools removed → forces plain-text summary
    end

    Groq-->>API: finish_reason: stop — final reply

    API->>DB: Save assistant reply
    API-->>B: All new messages

    alt show_metric called
        B->>User: 📊 Live widget card
    else preview_dashboard called
        B->>User: 🗂 Dashboard preview + Confirm / Discard
    else add_widget / remove_widget called
        B->>User: ✅ Success card + dashboard link
    else Text reply
        B->>User: 💬 Chat bubble in user's language
    end

Tool result is appended to the message thread → sent back to Groq → loop repeats up to 5 times → finish_reason: stop → plain-text summary returned to user. After round 1, tools are set to nil (forces summary, prevents infinite chains).

🧠 Context Injection

What's sentWhenWhy
System prompt (base) Every request Rules, tool usage guide, language behavior. Cache-stable — Groq caches this prefix.
System prompt (context extension) Only when a dashboard is open Preview-edit rules + live value instructions. Saves ~100 tokens on no-context requests.
Live widget values When a dashboard is open in the browser Model sees current sensor readings without a tool call — faster response for status questions.
Widget configs When a dashboard is open in the browser Model knows widget titles, types, and machine IDs — used for @Widget Title targeting.
Conversation history Every request Last 8 messages. Enables multi-turn edits ("change that to 200") without repeating the machine name.

🖥️ Response Types

📊
Live Metric Card
A gauge, KPI, or trend widget rendered in the chat canvas with real-time data. Clicking it keeps the widget visible.
Triggered by: show_metric
🗂️
Dashboard Preview
Full widget set rendered inline with Confirm and Discard buttons. No DB write until user confirms.
Triggered by: preview_dashboard, preview_add/update/remove_widget
✅
Success Card
Confirmation that a widget was added or removed, with a direct link to the updated dashboard.
Triggered by: add_widget_to_dashboard, remove_widget
💬
Text Bubble
Plain-text answer in the user's language. Used for greetings, trend summaries, alert lists, and production counts.
Triggered by: all read tools + finish_reason: stop with no widget

🔑 Key Design Decisions

DecisionReason
Max 5 Groq iterationsPrevents infinite loops. After round 1, tools are removed → LLM must summarize.
History capped at 8 messagesKeeps Groq prompt-cache prefix stable. Enough for multi-turn edits without context explosion.
Preview draft saved in DBSurvives page refresh. AI page restores in-progress dashboard on load.
Context cleared on AI page mountPrevents stale dashboard state leaking from previous navigation — fixed bug that added ~870 phantom tokens.
Zero-tool gate for greetingsIf no context + message has no action keywords → send no tools. Drops from 1143 to 273 prompt tokens.
