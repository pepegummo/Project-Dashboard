แสดงการสื่อสารระหว่าง User · Backend · Groq · Database · Model: openai/gpt-oss-20b (Groq prompt-cached)

Tools ที่ส่งให้ AI ทุกครั้ง (15 tools)

READ (7): get_machines · get_latest_telemetry · show_metric · get_telemetry_trend · get_active_alerts · get_daily_count · list_dashboards
PREVIEW (4): preview_dashboard · preview_add_widget · preview_remove_widget · preview_update_widget
WRITE (4): add_widget_to_dashboard · remove_widget · create_alert · manage_alert_event

✅ Prompt caching: backend ส่ง tools ครบ 15 ตัวทุก request แต่ prefix (system prompt + tool schemas) คงที่ → Groq cache ให้อัตโนมัติ: ลดค่า token 50% และไม่นับ rate limit (ไม่ต้องมี router/keyword)
⭐ show_metric (rule 11): ถาม/ขอ/เพิ่ม widget ของ metric ใด → AI map คำ (ภาษาใดก็ได้) → field key อังกฤษ; backend resolve machine+field → ส่ง widget spec; Frontend แสดง focus card (live, ยังไม่บันทึก — กด "Add to dashboard" เอง) หรือ highlight ถ้ามี widget นั้นอยู่แล้ว · ไม่ถามว่า "dashboard ไหน"
⚠ Tools filtered ตาม role เท่านั้น — admin/editor เห็นครบ 15; viewer เห็นเฉพาะ READ+PREVIEW
⚠ create_custom_dashboard ไม่อยู่ใน tool list เลย — Frontend เรียกตรงผ่าน POST /tools/execute หลัง user กด Confirm (AI สร้าง dashboard เองไม่ได้)
⚠ add_widget_to_dashboard ถ้าไม่ระบุชื่อ dashboard → backend gate ตอบกลับให้ใช้ show_metric แทน (กันไม่ให้ถาม "dashboard ไหน")
⚠ Machine resolve แบบ substring: ชื่อจริง "Checkweigher CW-01" แต่ user พิมพ์ "CW-01" (ไม่ใช่ exact match)
⚠ System prompt มี 11 rules — rule 10: ถ้ากำกวมจริง ๆ ("แก้ให้หน่อย") ถามกลับ 1 คำถาม · rule 11: เห็น/ขอ/เพิ่ม metric → show_metric
⚠ History ที่ส่ง Groq: last 8 messages เฉพาะ role=user/assistant เท่านั้น (tool messages ถูก strip ออก)
⚠ Context (preview บนจอ) ถูก append ท้ายสุด หลัง history → ไม่ทำลาย cached prefix
⚠ Loop สูงสุด 5 รอบ (tool_calls → execute → tool_calls → …) ก่อนหยุด

Case 1 — Simple Q&A ("hello" / ถามข้อมูลทั่วไป)

sequenceDiagram actor U as User participant FE as Frontend participant BE as Backend (Go) participant Groq as Groq API participant DB as Database U->>FE: พิมพ์ "hello" FE->>BE: POST /chat { message, conversationId } BE->>DB: INSERT ai_messages (role=user) BE->>DB: SELECT ai_messages (last 8, user+assistant only) note over BE: buildGroqTools(role)
→ 15 tools (admin/editor)
→ read+preview only (viewer)
prefix คงที่ → Groq cache hit BE->>Groq: POST /completions { messages, tools×15 } Groq-->>BE: finish_reason="stop" content="Hello!" BE->>DB: INSERT ai_messages (role=assistant) BE-->>FE: { messages } FE-->>U: แสดงข้อความ "Hello!"

⚠ 15 tools ถูกส่งไปด้วยแต่ model ไม่เรียกใช้ เพราะ system prompt rule 1: "Plain text for greetings — no tool call" · prefix ถูก cache ไว้แล้ว ค่า token จึงถูก (50% off)

Case 2 — เห็น/ขอ/เพิ่ม metric ("speed ของ CW-01" / "add weight widget") → show_metric

sequenceDiagram actor U as User participant FE as Frontend participant BE as Backend (Go) participant Groq as Groq API participant DB as Database U->>FE: "speed ของ CW-01 เท่าไหร่?" FE->>BE: POST /chat { message, conversationId } BE->>DB: INSERT ai_messages (role=user) BE->>DB: SELECT ai_messages (last 8, user+assistant only) note over BE: buildGroqTools(role) → 15 tools BE->>Groq: POST /completions { messages, tools×15 } note over Groq: rule 11 — เห็น/ขอ metric
map "speed"→field key, เลือก show_metric Groq-->>BE: finish_reason="tool_calls"
tool: show_metric({machine:"CW-01", metric:"speed"}) note over BE: resolveMachineID (substring) + lookup machine_fields BE->>DB: SELECT name, label, unit, min, max FROM machine_fields DB-->>BE: { label:"Belt Speed", unit:"ppm", min:0, max:120 } BE->>DB: INSERT ai_messages (role=tool, result={widget:{…}}) note over BE: callTools = nil (drop tools for summary call) BE->>Groq: POST /completions { messages + widget spec, no tools } Groq-->>BE: finish_reason="stop"
content="นี่คือ speed ของ CW-01 ครับ" BE->>DB: INSERT ai_messages (role=assistant) BE-->>FE: { messages (toolResult: {widget}) } note over FE: มี widget นี้บน dashboard → highlight
ไม่มี → focus card (live, ปุ่ม Add to dashboard) FE-->>U: แสดง focus card / highlight

Groq call 2 ครั้ง — ครั้งแรกเลือก show_metric, ครั้งสองสรุปสั้น ๆ (callTools=nil) · loop cap: 5 รอบ
Frontend อ่าน toolResult.widget ตรง ๆ — ไม่ต้องเดา machine/metric จากข้อความ · focus card เป็น ephemeral (ไม่บันทึกจนกว่าจะกด Add)
"add weight widget" (ไม่ระบุ dashboard) เข้า flow เดียวกัน — ไม่ใช่ add_widget_to_dashboard

Case 3 — สร้าง Dashboard ("create dashboard about CW-01")

sequenceDiagram actor U as User participant FE as Frontend participant BE as Backend (Go) participant Groq as Groq API participant DB as Database U->>FE: "create dashboard about CW-01" FE->>BE: POST /chat { message, conversationId } BE->>DB: INSERT ai_messages (role=user) BE->>DB: SELECT ai_messages (last 8, user+assistant only) note over BE: buildGroqTools(role) → 15 tools
(create_custom_dashboard ไม่อยู่ใน list) BE->>Groq: POST /completions { messages, tools×15 } note over Groq: ชื่อ CW-01 ชัดเจน (rule 2)
เลือก preview_dashboard Groq-->>BE: finish_reason="tool_calls"
tool: preview_dashboard({machine:"CW-01", template:"machine_overview"}) note over BE: Preview — ไม่เขียน DB BE->>DB: SELECT machine_fields WHERE machine=CW-01 DB-->>BE: fields: [speed, temp, vibration] BE->>DB: INSERT ai_messages (role=tool) note over BE: callTools = nil (drop tools) BE->>Groq: POST /completions { messages + preview result, no tools } Groq-->>BE: finish_reason="stop"

BE->>DB: INSERT ai_messages (role=assistant) BE-->>FE: { messages, toolResult: {widgets:[...]} } FE->>BE: PUT /preview-draft { conversationId, data: preview } BE->>DB: UPSERT ai_preview_drafts FE-->>U: แสดง preview panel + ปุ่ม Confirm note over U,DB: User กดปุ่ม Confirm U->>FE: กด Confirm FE->>BE: POST /tools/execute { toolName:"create_custom_dashboard", params:{...} } BE->>DB: INSERT dashboards BE->>DB: INSERT dashboard_widgets ×4 DB-->>BE: { dashboardId } BE-->>FE: { success, data: { dashboardId } } FE->>BE: DELETE /preview-draft BE->>DB: DELETE ai_preview_drafts FE-->>U: redirect → /dashboards/:id

create_custom_dashboard ไม่ได้อยู่ใน AllTools() เลย — Frontend เรียกตรงผ่าน POST /tools/execute หลัง user กด Confirm เท่านั้น
Groq call 2 ครั้ง — ครั้งแรก model เลือก preview_dashboard, ครั้งสองสรุปผล (callTools=nil)

Case 4 — แก้ Widget ใน Preview ("change metric to temperature")

sequenceDiagram actor U as User participant FE as Frontend participant BE as Backend (Go) participant Groq as Groq API participant DB as Database note over U,DB: มี preview dashboard อยู่บนหน้าจอแล้ว U->>FE: "change metric to temperature" FE->>BE: POST /chat { message, conversationId, context: previewData } BE->>DB: INSERT ai_messages (role=user) BE->>DB: SELECT ai_messages (last 8, user+assistant only) note over BE: context != "" → append system msg
"Authoritative current dashboard state…"
buildGroqTools(role) → 15 tools BE->>Groq: POST /completions { messages + context, tools×15 } note over Groq: มี preview context → เห็น widget ปัจจุบัน
เลือก preview_update_widget Groq-->>BE: finish_reason="tool_calls"
tool: preview_update_widget({widget_title:"Trend", metric:"temperature"}) note over BE: ไม่เขียน DB — แค่ return changes BE-->>BE: PreviewUpdateWidget() → {updated:true, changes:{metric:"temperature"}} BE->>DB: INSERT ai_messages (role=tool) note over BE: callTools = nil BE->>Groq: POST /completions { messages + changes result, no tools } Groq-->>BE: finish_reason="stop"
content="อัปเดต metric เป็น temperature แล้วครับ" BE->>DB: INSERT ai_messages (role=assistant) BE-->>FE: { messages, toolResult: {changes} } FE->>FE: apply changes ใน preview state (local) FE->>BE: PUT /preview-draft { data: updated preview } BE->>DB: UPSERT ai_preview_drafts FE-->>U: preview อัปเดตแล้ว

preview_update_widget ไม่ query DB เลย — validate แล้ว return changes กลับมา Frontend apply ใน local state เอง
context injection: ถ้า FE ส่ง context มา BE จะ append เป็น system message สุดท้ายก่อนส่ง Groq (recency wins)
