IotVision AI — Sequence Diagram

แสดงการสื่อสารระหว่าง User · Backend · Groq · Database · Model: qwen/qwen3-32b

Case 1 — Simple Q&A ("hello" / ถามข้อมูลทั่วไป)

sequenceDiagram actor U as User participant FE as Frontend participant BE as Backend (Go) participant Groq as Groq API participant DB as Database U->>FE: พิมพ์ "hello" FE->>BE: POST /chat { message, conversationId } BE->>DB: INSERT ai_messages (role=user) BE->>DB: SELECT ai_messages (history ≤8 msgs) note over BE: hint scan: ไม่เจอ action verb
includeBuilder = false
ส่ง 6 READ tools BE->>Groq: POST /completions { messages, tools×6 } Groq-->>BE: finish_reason="stop" content="Hello!" BE->>DB: INSERT ai_messages (role=assistant) BE-->>FE: { messages } FE-->>U: แสดงข้อความ "Hello!"

⚠ 6 READ tools ถูกส่งไปด้วยแต่ model ไม่เรียกใช้ เพราะ system prompt rule 1: "Plain text for greetings — no tool call"

Case 2 — ถามข้อมูล + ต้องใช้ Tool ("speed ของ CW-01 เท่าไหร่?")

sequenceDiagram actor U as User participant FE as Frontend participant BE as Backend (Go) participant Groq as Groq API participant DB as Database U->>FE: "speed ของ CW-01 เท่าไหร่?" FE->>BE: POST /chat { message, conversationId } BE->>DB: INSERT ai_messages (role=user) BE->>DB: SELECT ai_messages (history) note over BE: hint scan: ไม่เจอ action verb
includeBuilder = false → 6 READ tools BE->>Groq: POST /completions { messages, tools×6 } note over Groq: รู้ชื่อ CW-01 ชัดเจน (rule 2)
เลือก get_latest_telemetry Groq-->>BE: finish_reason="tool_calls"
tool: get_latest_telemetry("CW-01") BE->>DB: SELECT telemetry_raw WHERE machine=CW-01 DB-->>BE: { speed: 1200, timestamp: ... } BE->>DB: INSERT ai_messages (role=tool) note over BE: callTools = nil (drop tools) BE->>Groq: POST /completions { messages + tool result } Groq-->>BE: finish_reason="stop"
content="ความเร็วของ CW-01 ตอนนี้ 1,200 rpm" BE->>DB: INSERT ai_messages (role=assistant) BE-->>FE: { messages } FE-->>U: แสดงคำตอบ

Groq call 2 ครั้ง — ครั้งแรก model เลือก tool, ครั้งสองสรุปผลเป็นข้อความ

Case 3 — สร้าง Dashboard ("create dashboard about CW-01")

sequenceDiagram actor U as User participant FE as Frontend participant BE as Backend (Go) participant Groq as Groq API participant DB as Database U->>FE: "create dashboard about CW-01" FE->>BE: POST /chat { message, conversationId } BE->>DB: INSERT ai_messages (role=user) BE->>DB: SELECT ai_messages (history) note over BE: hint scan: "create" match ✓
includeBuilder = true → 14 tools BE->>Groq: POST /completions { messages, tools×14 } note over Groq: ชื่อ CW-01 ชัดเจน (rule 2)
เลือก preview_dashboard Groq-->>BE: finish_reason="tool_calls"
tool: preview_dashboard({machine:"CW-01", template:"machine_overview"}) note over BE: Preview — ไม่เขียน DB BE->>DB: SELECT machine_fields WHERE machine=CW-01 DB-->>BE: fields: [speed, temp, vibration] BE->>DB: INSERT ai_messages (role=tool) note over BE: callTools = nil (drop tools) BE->>Groq: POST /completions { messages + preview result } Groq-->>BE: finish_reason="stop"
content="นี่คือ preview dashboard ครับ กดยืนยัน..." BE->>DB: INSERT ai_messages (role=assistant) BE-->>FE: { messages, toolResult: {widgets:[...]} } FE->>BE: PUT /preview-draft { data: preview } BE->>DB: UPSERT ai_preview_drafts FE-->>U: แสดง preview panel + ปุ่ม Confirm note over U,DB: User กดปุ่ม Confirm U->>FE: กด Confirm FE->>BE: POST /tools/execute { toolName:"create_custom_dashboard", params:{...} } BE->>DB: INSERT dashboards BE->>DB: INSERT dashboard_widgets ×4 DB-->>BE: { dashboardId } BE-->>FE: { success, dashboardId, url } FE->>BE: DELETE /preview-draft BE->>DB: DELETE ai_preview_drafts FE-->>U: redirect → /dashboards/:id

Groq call 2 ครั้ง — create_custom_dashboard ไม่ได้อยู่ใน tools ที่ส่งให้ AI เลย Frontend เรียกตรงผ่าน /tools/execute หลัง user กด Confirm เท่านั้น

Case 4 — แก้ Widget ใน Preview ("change metric to temperature")

sequenceDiagram actor U as User participant FE as Frontend participant BE as Backend (Go) participant Groq as Groq API participant DB as Database note over U,DB: มี preview dashboard อยู่บนหน้าจอแล้ว U->>FE: "change metric to temperature" FE->>BE: POST /chat { message, conversationId, context: previewData } BE->>DB: INSERT ai_messages (role=user) BE->>DB: SELECT ai_messages (history) note over BE: hint scan: "change" match ✓
+ context มีค่า → includeBuilder = true → 14 tools BE->>Groq: POST /completions { messages, context, tools×14 } note over Groq: มี preview context → เห็น widget ปัจจุบัน
เลือก preview_update_widget Groq-->>BE: finish_reason="tool_calls"
tool: preview_update_widget({widget_title:"Trend", metric:"temperature"}) note over BE: ไม่เขียน DB — แค่ return changes BE-->>BE: PreviewUpdateWidget() → {updated:true, changes:{metric:"temperature"}} BE->>DB: INSERT ai_messages (role=tool) BE->>Groq: POST /completions { messages + changes result } Groq-->>BE: finish_reason="stop"
content="อัปเดต metric เป็น temperature แล้วครับ" BE->>DB: INSERT ai_messages (role=assistant) BE-->>FE: { messages, toolResult: {changes} } FE->>FE: apply changes ใน preview state (local) FE->>BE: PUT /preview-draft { data: updated preview } BE->>DB: UPSERT ai_preview_drafts FE-->>U: preview อัปเดตแล้ว

preview_update_widget ไม่ query DB เลย — แค่ validate แล้ว return changes กลับมา Frontend เป็นคน apply ใน local state เอง
